Cybersecurity

SWOT · STRIDE · OWASP · Compliance · Roadmap

Campus Buddy's core attack surface: identity trust (who is really a student?) and data sensitivity (location + social graph of young adults). Security is a prerequisite for university procurement approval.

Cybersecurity · 1 of 5

Security SWOT Analysis

✚ Helpful
✖ Harmful
Internal
Strengths
  • · Stateless JWT — reduced session attack surface
  • · React frontend: no direct DB access from client
  • · REST API cleanly separates frontend from data
  • · AWS Shield Standard: built-in DDoS protection
  • · CDN-served frontend: zero server-side exec risk
Weaknesses
  • · Student PII (names, majors, location) is high-value
  • · WebSocket layer harder to secure than HTTP
  • · MVP: no rate limiting or input sanitization yet
  • · University SSO — auth failures outside our control
  • · Small team = limited dedicated security capacity
External
Opportunities
  • · Univ. IT partnerships: SIEM + compliance guidance
  • · FERPA/GDPR compliance as sales differentiator
  • · University SSO removes password management risk
  • · Bug bounty: crowdsourced testing at low cost
  • · SOC 2 / ISO 27001 unlocks enterprise contracts
Threats
  • · Students: weak passwords, phishing, credential reuse
  • · Social platforms targeted for stalking + doxxing
  • · Fake student profiles to harvest data or harass
  • · Location + messaging = serious physical safety risk
  • · API scraping could expose entire activity graph

STRIDE Threat Model (Microsoft Framework)

S
Spoofing Identity
Fake student accounts impersonating real users · Risk: social engineering + physical harm · Severity: High
→ Mitigations: .edu email verification · University SSO as primary auth · Rate-limit account creation · Profile flagging + fast takedowns
T
Tampering with Data
Forged API requests modify others' posts or messages · Severity: Medium
→ Mitigations: JWT userId ownership checks on all writes · DB-level row constraints · Append-only immutable message logs
R
Repudiation
User denies sending harassment — no audit trail = no moderation · Severity: Medium-High
→ Mitigations: Immutable server-side audit logs · AWS CloudWatch / S3 object lock · Content reporting system with case tracking
I
Information Disclosure
IDOR exposes PII — FERPA + GDPR violation risk · Severity: High
→ Mitigations: Strict API serialization · IDOR permission checks · AES-256 at rest · TLS 1.3 + HSTS · No message content in app logs
D
Denial of Service
Bot floods feed API or WebSocket — damages UX + compute cost · Severity: Medium
→ Mitigations: Rate limiting 60 req/min JWT + IP · AWS WAF + Shield Standard · Max 3 WebSocket sessions/user
E
Elevation of Privilege
Student forges JWT claims to gain admin access · Severity: Critical
→ Mitigations: RS256 asymmetric JWT signing · Server-side RBAC on every request · Admin API on separate internal network · Principle of least privilege on AWS IAM

Compliance Requirements

🇺🇸
FERPA
Family Educational Rights & Privacy Act
  • · Applies to all US university deployments
  • · Campus Buddy = "school official" under FERPA
  • · Students must access, correct & delete their data
  • · No sharing education records without consent
  • → Action: Privacy Policy + complete data deletion (not soft-delete)
🇪🇺
GDPR
General Data Protection Regulation
  • · Applies to any EU-based student regardless of server location
  • · Explicit consent for data collection
  • · Right to erasure ("right to be forgotten")
  • · Data minimization + 72-hour breach notification
  • → Action: Cookie banner · DPA template · AWS eu-west region for EU campuses
SOC 2 Type II
Unlocks enterprise university contracts
ISO 27001
International security certification
HackerOne Bug Bounty
Crowdsourced testing at low cost
